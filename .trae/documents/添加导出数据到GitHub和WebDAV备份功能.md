# 添加导出数据到GitHub和WebDAV备份功能

## 功能概述
为Todo应用添加两种新的备份方式：
1. 导出数据到GitHub仓库
2. 导出数据到WebDAV服务器

## 技术实现方案

### 1. UI修改
- 在现有的导出/导入按钮旁边添加新的备份按钮
- 添加配置对话框，用于设置GitHub和WebDAV的连接信息
- 添加状态提示和错误处理

### 2. GitHub集成
- 使用GitHub API创建或更新仓库中的备份文件
- 实现OAuth认证流程获取访问令牌
- 存储用户配置（仓库名称、文件路径等）
- 处理API错误和重试机制

### 3. WebDAV集成
- 实现WebDAV客户端功能，支持文件上传
- 添加服务器配置选项（URL、用户名、密码）
- 处理认证和连接错误
- 支持自定义备份路径

### 4. 数据处理
- 复用现有的数据序列化逻辑
- 添加备份时间戳和版本信息
- 实现增量备份和完整备份选项

### 5. 安全考虑
- 安全存储认证令牌和密码
- 实现令牌过期处理和刷新
- 添加备份成功/失败的通知

## 实现步骤
1. 创建备份配置对话框和UI元素
2. 实现GitHub OAuth认证流程
3. 实现GitHub API文件操作功能
4. 实现WebDAV客户端功能
5. 集成到现有的导出功能中
6. 测试和优化用户体验

## 预期效果
- 用户可以一键备份任务数据到GitHub仓库
- 用户可以一键备份任务数据到WebDAV服务器
- 支持配置多个备份目标
- 提供备份状态和历史记录

## 注意事项
- 确保OAuth流程的安全性
- 处理网络错误和离线情况
- 提供清晰的用户提示和错误信息
- 尊重API速率限制和服务器资源
